== Fixed issues

The following issues are fixed for this release:

* Before this update, incorrect subnet was association with automatic v-net subnet creation. As a consequence, previously configured subnets were affected, leading to Azure installation failure. In this release, correct subnet assignment is done in pre-configured v-nets. As a result, the Azure installation issue is resolved. (link:https://issues.redhat.com/browse/OCPBUGS-73781[OCPBUGS-73781])

* Before this update, CoreOS networking timeout occurred during automation of agent-based {product-title} installations on VLAN interfaces in bare metal environments. As a consequence, CoreOS nodes failed to automatically configure networking, preventing installation on bare metal systems. With this release, the `pre-network-manager-config` service timeout is increased, resolving the networking issue in {product-title} installations. As a result, during agent-based {product-title} installation, CoreOS nodes now automatically configure and start networking, eliminating the need for manual intervention. (link:https://issues.redhat.com/browse/OCPBUGS-74940[OCPBUGS-74940])

* Before this update, the `collect-profiles` job was affected by maintenance difficulties, causing confusion for customers and support engineers during troubleshooting. With this release, the `collect-profiles` job is removed, improving user experience and reducing maintenance effort. (link:https://issues.redhat.com/browse/OCPBUGS-76587[OCPBUGS-76587])
